# 03 · Guardrails(護欄:讓 AI 不要掉下去)

**什麼時候用:** 你同意 AI 動手了,但要它「只做該做的」,不要順手亂改。

**常用護欄句型(挑需要的貼):**

```
請只修改指定檔案。
請不要新增套件。
請不要重構整個專案。
請不要新增登入、後台、資料庫。
請不要修改不相關區塊。
請修改前先說明。
```

**同意修改時,這樣講:**

```
同意,請照剛剛的計畫修改。

請只修改必要部分。
完成後請告訴我:
1. 改了哪個檔案
2. 改了什麼
3. 怎麼驗收
```

> 改完一定要自己看:`npm run dev` 看畫面、`git status` 看動了哪些檔、`git diff` 看實際改了什麼。
