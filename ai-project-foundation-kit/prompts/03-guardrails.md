# 03 · Implementer 與護欄(同意動手之後)

**什麼時候用:** planner 的計畫你人審過了,現在讓 AI 照計畫動手 —— 但要它「只做該做的」。

**同意修改時,複製這段:**

```
你現在扮演 implementer。
請依照「剛剛已核准的計畫」實作。

限制:
1. 只能修改:<允許檔案清單>
2. 不要新增套件
3. 不要改 package.json
4. 不要重構
5. 如果超出原計畫,先停下來回報,不可直接修改

完成後請固定回報:
A. 實際修改的檔案
B. 每個檔案改了什麼
C. 哪裡對應到資料合約
D. 我現在要怎麼手動驗收
E. 哪些地方你沒有改
```

**常用護欄句型(需要就多貼幾句):**

```
請只修改指定檔案。
請不要新增套件。
請不要重構整個專案。
請不要新增登入、後台、資料庫。
請不要修改不相關區塊。
請修改前先說明。
```

**AI 越界時的救援:**

```
請停止目前修改。
請列出你這次動過的所有檔案。
超出允許清單的部分,請說明原因,並等我決定是否還原。
```

> AI 回報的 D(如何手動驗收)就是你下一步要做的事:`npm run dev` 看畫面、`git status` 看動了哪些檔、`git diff` 看實際改了什麼。
> 它做完不算完成,你照 D 驗收過才算完成。
