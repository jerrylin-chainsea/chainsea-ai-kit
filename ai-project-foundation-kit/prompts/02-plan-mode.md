# 02 · Planner(先計畫,再動手)

**什麼時候用:** 你要改東西,但要先看 AI 打算怎麼改、會不會改太多。每一次動手前都先跑這張。

**複製下面這段(`<>` 換成當堂任務):**

```
你現在扮演 planner。
先不要改檔,也不要執行寫入動作。

請先閱讀:
- AGENTS.md
- CLAUDE.md
- data-lab/report.json
- 這次任務相關的檔案

任務:
我要完成「倉儲營運 Dashboard + LINE OA Flex 推播」中的下一步:<這堂指定任務>

限制:
1. 只能修改:<允許檔案清單>
2. 不要新增套件
3. 不要改 package.json
4. 不要重構
5. 如果需要新增檔案,先說明原因,不可直接做

請只用以下格式回答:
A. 你準備修改的檔案
B. 每個檔案各改什麼
C. 資料合約會如何被使用
D. 驗收方式
E. 可能風險
F. 哪一步需要人類拍板
```

**拿到計畫後,先問四題再決定放不放行:**

1. 可修改檔案對嗎?(在允許清單內)
2. 有沒有新套件?
3. 有沒有改 package.json?
4. 有沒有重構傾向?

> 計畫看起來想改太多,就把它拉回來:「請縮小範圍,這次只修改 <允許檔案>,其他都不要動。」
> 固定格式不是龜毛 —— A 到 F 每次都長一樣,你才對得快、放行得安心。
