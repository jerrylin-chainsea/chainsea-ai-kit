# M14 學員工作手冊｜照著做就會的逐案例路徑

> 給**學生自己照著走**。每個案例:**目標 → 複製這段貼上 / 動作 → 你應該看到 → 沒看到怎麼辦**。
> 配合投影片(`slides/u14-c1-normalday`、`slides/u14-c2-incident`);投影片講「為什麼」,這份給你「照著做」。
> 老師主持用的事件卡在同資料夾的 `00`~`06` 那幾張卡;這份是**給你照著操作的版本**。

**M14 不教新工具。** coding agent、M12 決策助理,都是你前面學過的。M14 只是把它們**開起來當一間公司營運**:跑正常的一天、處理出包、寫一頁交接。你只要看三個地方:左邊**檔案樹**、跟 AI 講話的**對話框**、跑程式的**終端機**。

> 心法:**營運不是一個功能,是把所有功能串起來,每天跑一遍。**

---

## 課前(很重要,先讀):從零也能上手,不用補完舊作業

你前面有些單元沒做完?**沒關係。** M14 練的是「把流程跑一遍」,不是補完所有舊作業。落後的部分**用老師的 M12 starter 補位**,從乾淨起點接手就好。

**這份手冊預設你手上只有 M12 starter,其他通通可以用講的補。** 對照表:

| 投影片/事件卡提到 | 你沒做過?這樣補位 |
|---|---|
| M12 決策助理(三角色) | 用 `m12-crew-starter/`,跑 `python run.py` 就有 |
| 銷售/營運資料 | 用 M12 的 `m12-crew-starter/data/sales_sample.csv` 範例資料 |
| M13 發布站、社群、LINE OA | **用講的描述即可**:「假設我網站首頁有一塊公告區」,把公告貼上去就好,沒站台不影響演練 |
| 庫存表、叫料、n8n 自動化(U5–U9) | **用講的描述即可**:口頭說「某品項剩 8%」就是一個缺料事件,不需要真的庫存系統 |

> 一句話:**有 M12 starter 就能全程跑完。** 其他缺的,演練時用嘴巴補,不要卡在「我還沒建那個系統」。

# 課 1 ｜ 跑一次正常營業日

> 投影片 `u14-c1-normalday`。今天像開店一樣,跑一個正常的營業日:換成你的店 → 畫營運地圖 → 跑一天(開店→尖峰→老闆問→收店)→ 看一年史。

## 案例 1 ｜ 5 分鐘:把範例換成你自己的店(手動,先有感)

**目標:** 範例是「大甲鍋貼」,但要有感,得是**你自己的店**。先花 5 分鐘換成你的,演練才投入。

**① 動作(自己做,不用 prompt,每步約 1 分鐘):**
1. **取個品牌名**:把「大甲鍋貼」換成你的(例:喵星人貓糧、阿明早餐車)。寫下來:`____________`
2. **一句話業務**:你賣什麼、賣給誰?例:「賣給上班族的高蛋白貓糧訂閱」。`____________`
3. **挑一個你最在意的指標**:例「外送佔比」「缺貨次數」「客單價」。演練時就盯它。`____________`

**② 順手把 M12 的資料也換成你的(複製這段,貼到對話框):**
```text
請打開 m12-crew-starter/data/sales_sample.csv,
把裡面 1 到 2 個品項換成我的商品(我的店是:____,我賣:____)。
其他數字、欄位、逗號都不要動,只改品項名稱。改完存檔。
```

**✓ 你應該看到:** 你心裡有一個清楚的「我的店」,而且能說出「我最在意的數字是 ____」。M12 的 csv 裡也出現了你的品項。

**✗ 沒看到 / 卡住:**
- 沒空換 csv → 跳過第②步也行,用範例品項先把流程跑通,品牌名在心裡換掉就好。
- csv 改壞了(那行亂掉)→ 多半是逗號被刪到,叫它「把那一行還原成原本格式,只換品項名」。

> 不用做得漂亮,**能辨識「這是我的公司」就好**。美化是課後的事。

## 案例 2 ｜ 請 AI 幫你畫一張營運地圖(AI 帶做)

**目標:** 把你做過的東西,整理成一張「**資料怎麼流**」的地圖。像店裡的動線圖:客人從哪進來、單往哪送、廚房怎麼出餐。我們畫的是**資料**的動線。看得懂資料怎麼流,你才知道哪裡斷了要去哪修。

**① 複製這段,貼到對話框,按 Enter:**
```text
幫我把我的系統整理成一張營運地圖,分三層:
對外層(網站/社群/客服)、內部營運層(庫存/流程/叫料)、資料與決策層(M12 助理 + 營運資料表)。
標出資料從顧客到 AI 建議,是怎麼一步步流過去的。
我目前只有 M12 決策助理,其他層先用文字標「規劃中」即可。
```

**✓ 你應該看到:** 它回一張分三層、標了資料流向的地圖。順著箭頭走一次就讀懂了,大概像這條線:
```
對外    顧客下單(網站/外送)
          │
內部    進到營運資料表
          │
決策    M12 讀資料、抓異常
          │
你      看建議、做決定
```

**✗ 沒看到 / 卡住:**
- 它畫得太複雜 → 跟它說「**只畫會用到的節點**,我只有 M12,其他先省略」。
- 看不懂 → 你不用看每個細節,**順著箭頭講一次**:顧客 → 資料 → AI → 你。能講出這條線就過了。

## 案例 3 ｜ 跑一次正常營業日(AI 帶做,M14 必做整合演練)

**目標:** 情境是「新品促銷日」。照一個營業日的順序跑一次:**開店公告 → 午餐尖峰缺料 → 老闆問生意(叫 M12 判讀)→ 收店交接摘要**。一個營業日,就把前面所有東西接成一條線。

> 沒有真網站、真庫存系統?**全部用講的補位**:公告貼哪裡就講「貼在我網站公告區」,缺料就口頭說「某品項剩 8%」。M12 那步是真的跑,其他用嘴巴串起來即可。

### 第 1 步 · 開店:請 AI 寫新品公告

**複製這段,貼到對話框:**
```text
今天推新品「泡菜鍋貼」(你的新品:____)。
幫我寫一則開賣公告,放在我的網站跟社群,
語氣熱鬧、有食慾,附一句限時優惠。
```
**✓ 你應該看到:** 拿到一則能直接貼的新品公告。把它「貼到」你的網站公告區(沒站台就口頭說「貼上去了」)= 開店這步完成。

### 第 2 步 · 午餐尖峰:缺料提醒會響(先記下來,別處理)

**動作(用講的即可):** 新品太受歡迎,某個料快用完。這時假設系統跳一個提醒:
```text
⚠ 泡菜內餡 剩餘 8%
   依目前售速,約 40 分鐘用完
```
**✓ 你應該看到:** 你**記下這個缺料事件**,先不處理。這是**下一堂「處理異常」的伏筆**,先放著。

### 第 3 步 · 下午:老闆問生意,叫 M12 判讀當日(這步是重點)

**先確認 M12 能跑** ： 在終端機:
```bash
cd m12-crew-starter
python run.py
```
**再複製這段,貼到對話框,叫 M12 判讀今天:**
```text
用今天(促銷日)的銷售資料 data/sales_sample.csv,幫我判讀:
今天生意如何?有沒有哪裡要注意?
請用 insight / anomalies / risk_level / action_items 四欄回我。
```
**✓ 你應該看到:** M12 給出當日摘要,還是你熟悉的那四欄,大概像這樣:
```
insight: "新品帶動整體 +18%"
anomalies: ["泡菜內餡快缺料"]
risk_level: "中"
action_items: ["下午補叫泡菜料"]
```
這就是 **M12 在 M14 裡的用法**:每天幫你把當日資料看一遍。

**✗ 沒看到 / 卡住:**
- `run.py` 報錯 / 抓不到金鑰 → 回 `m12-crew-starter/README.md` 的課前準備,金鑰那三步漏一步就抓不到。
- 沒有「今天促銷日」的資料 → 直接用 `data/sales_sample.csv` 範例資料當「今天」,一樣能判讀。

### 第 4 步 · 收店:請 AI 整理可交接的當日摘要

**複製這段,貼到對話框:**
```text
幫我把今天的營運整理成一段交班摘要:
今天賣得如何、發生什麼狀況(含那個缺料提醒)、明天要注意什麼。
三五句、口語,讓接班的人一看就懂。
```
**✓ 你應該看到:** 拿到一段能傳給接班人的當日摘要 = 收店完成。

### 收店前 · 記這四件事(下一堂會用到)

| 要記 | 問自己 |
|---|---|
| **成功** | 哪條流程跑通了? |
| **斷點** | 哪裡卡住、斷掉? |
| **人工接手** | 哪裡一定要人介入? |
| **資料差異** | 系統數字和實際對不對得上? |

**✓ 成功:** 四步跑完、四件事記下來 = 你把學過的全部串成一條線了。

**✗ 卡住:**
- 沒有完整資料可跑 → **用 M12 starter 的範例資料**,一樣能把流程走一遍。
- 某一步接不起來 → **記到「斷點」那欄**,這正是下一堂要練的處理,不用現在修。

# 課 2 ｜ 三輪異常 + 一頁交接

> 投影片 `u14-c2-incident`。開店一定會出狀況。今天用一套**五步框架**加上 AI,把三種突發狀況處理掉,最後寫一頁交接,完課。

## 心法:出事先「止血」,不是先找原因

像急診:病人大出血,醫生**先止血**,不會站著研究「他為什麼受傷」。原因很重要,但**晚一點再查**。止血不是亂修,是**先讓錯誤停止擴大**(暫停發布、接單、通知)。很多災難,都是「站著查原因」時損失一直變大。

---

## 五步框架(每個異常都跑這五步)

```
① 止血     先停止錯誤擴大(暫停發布/回覆/接單/通知),不是先找原因
② 定範圍   多少顧客 / 訂單 / 資料受影響?影響對外、金錢、還是內部?
③ 定層級   問題在哪一層:資料 / 流程 / 自動化 / Agent / 介面?
④ 決策     修正 or 採備援 or 人工接手?(小修正現場 vibe-code,大功能列課後)
⑤ 重測+紀錄 從問題前一站重測;寫下事件、決策、溝通、人工接手
```

**把順序記死:止血永遠第一,找原因排到第三步。** 因為你在查原因的每一分鐘,錯誤都還在繼續擴大。

**每一步怎麼用 AI(這才是 M14 的重點):**

| 步驟 | 用 AI 怎麼做 |
|---|---|
| ① 止血 | 問 AI:要關哪個開關、註解哪一段,比自己翻檔快 |
| ② 定範圍 | 用 M12 助理讀當下資料,估有多少訂單 / 顧客受影響 |
| ③ 定層級 | 把錯誤訊息 / log 貼給 AI,請它判斷是資料、流程還是程式問題 |
| ④ 決策 | 用 AI 起草對外說明、整理選項;**人做最終決定** |
| ⑤ 重測 | 請 AI 幫忙跑一次、讀結果確認 |

> 紅線:**AI 幫你「更快做到」,但「喊停」跟「拍板」,永遠是人。** 處理事件會動到錢、動到顧客,要不要止血、要不要對外講,必須是人按下去。

---

## 三輪異常(每輪:先止血做什麼 → 複製 prompt → 你應該看到)

> 三輪涵蓋三個出包來源:**對外品牌、營運資料、AI 自己**。練過這三種,大部分狀況你都接得住。
> 沒有真網站/真後台也能做:**止血用講的**(「我把那頁撤下了」),AI 那步是真的貼 prompt 處理。

### 第一輪 · 對外品牌:價格標錯

**目標:** 網站把新品價格標成 $5(應為 $50),已經掛了幾小時。

**① 先止血:** 把錯價那頁**撤下或改回**(沒站台就口頭說「我先把那則公告撤下了」)。

**② 複製這段,貼到對話框:**
```text
我們網站把一個品項價格標錯,已經掛了幾小時。
1. 幫我估算大概影響多少訂單(用今天的資料 data/sales_sample.csv)。
2. 幫我起草一則對顧客的更正說明,語氣誠懇。
我會自己確認後才發出。
```

**✓ 你應該看到:** 它做了五步裡的兩步 ：「② 定範圍」估了影響多少訂單、「③④ 決策+溝通」起草了更正說明。**先撤頁 → AI 估影響並起草 → 你確認後發出** = 第一輪處理完。

**✗ 卡住:**
- 不知道怎麼止血 → 記住一句:**先讓它停**(撤頁 / 設售完 / 暫停採用)。
- 它起草的說明太官腔 → 跟它說「**更口語、像老闆親自道歉**」,它會重寫。

### 第二輪 · 營運資料:爆單缺料

**目標:** 某品項暴單、庫存歸零,系統還在接單。

**① 先止血:** 把該品項先設**「售完 / 停用」**(用講的:「我先把泡菜鍋貼設成售完」)。

**② 複製這段,貼到對話框:**
```text
某品項今天暴單、庫存歸零。
1. 用今天資料 data/sales_sample.csv 看是不是只有這個品項、這個通路爆。
2. 幫我判斷:該暫停接單、還是先加叫料?給我依據。
我來做最後決定。
```

**✓ 你應該看到:** 它**給選項加依據,不替你拍板**。類似:「只有泡菜這品項爆、別的正常;1 小時內補得到料 → 建議加叫;補不到 → 建議暫停接單避免客訴。」**選哪個,你決定。**

**✗ 卡住:** 它直接幫你決定了 → 提醒它「**只給我選項跟依據,最後決定我來下**」。

### 第三輪 · AI 自己:建議失準

**目標:** M12 根據錯誤的資料範圍,給了一個怪怪的建議。

**① 先止血:** **暫停採用**這個 AI 建議,先不照它做。

**② 複製這段,貼到對話框:**
```text
你剛剛的建議,我覺得怪怪的。
1. 請列出你這個結論的「依據」是什麼。
2. 檢查你用的資料範圍,有沒有缺漏或被汙染。
沒有依據的建議,我不會採用。
```

**✓ 你應該看到:** 這正是 M12 課4 教的把關 ： **要求附依據、檢查資料範圍**。常常一問,就發現它用到了缺漏或汙染的資料,那這個建議就**不能信**。連 AI 自己出包,你也有方法接住。

**✗ 卡住:** 它硬要你相信 → 重複那句:「**沒有依據,我不採用**」,讓它把依據一條條列出來再說。

### 每處理完一張卡 · 記四件事

| 要記 | 寫什麼 |
|---|---|
| **事件** | 發生了什麼 |
| **決策** | 你決定怎麼做 |
| **溝通** | 對外講了什麼 |
| **人工接手** | 哪裡人介入了 |

**✓ 成功:** 三張卡都**先止血 → 再用 AI 處理 → 各留一筆紀錄**。

## 一頁交接(M14 的核心產出)

**目標:** 把今天的處理紀錄,用 AI 整理成一頁交接。寫下來,這套東西才是「公司的」,不是只有你能跑。

**① 複製這段,貼到對話框:**
```text
把我今天的處理紀錄,整理成一頁交接說明,包含:
1. 端到端流程(顧客到 AI 建議怎麼流)
2. AI 在哪裡幫忙、什麼情況不該信它
3. 人工必須接手的地方
4. 我們提煉的治理規則(3 到 5 條,例:改價需人工核准、庫存歸零停接單、AI 建議需附依據才採用)
```
