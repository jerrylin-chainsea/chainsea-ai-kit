# M14 ｜ AI 營運控制塔實戰 — 演練教具

這是一套「桌遊式」營運演練教具，給講師主持 M14（8hr / 2 次課）。**不另教新工具**（coding agent、M12 助理都是前面學過的，M14 只重用），把學員前面做出來的成果（M12 決策助理、M13 發布站、U5–9 系統）當成一間 AI 公司，**用 AI 來營運、處理異常、交接**。

## 怎麼用

| 檔案 | 用途 | 在哪一節 |
|---|---|---|
| `00-個人化腳本.md` | 用 starter 補位的組，5 分鐘把品牌換成自己的，重建所有權感 | 課1 開場 |
| `01-營運地圖模板.md` | 盤點全公司系統與角色，建立共同視圖 | 課1 |
| `02-正常情境腳本.md` | 跑第一次正常營業日 | 課1 |
| `03-異常事件卡.md` | 三輪異常事件投入 | 課2 |
| `04-角色卡.md` | 團隊分工 | 課1/課2 |
| `05-五步異常處理卡.md` | 異常處理決策框架 | 課2 |
| `06-一頁式交接範本.md` | 最終成果交接 | 課2 收尾 |

## 主持原則（重要）

- **這是「用 AI 營運」的演練，不是純人工桌遊。** 每個動作都示範怎麼用 AI 去做：用 M12 決策助理判讀資料、用 coding agent 讀錯誤與 vibe-code 小修正、用 AI 起草對外說明。M14 學的是**用 AI 處理營運與異常**的能力（見 `05-五步異常處理卡.md` 的「每一步怎麼用 AI」）。
- **修正分級**：現場可用 coding agent vibe-code 小繞道/小修正（老師在旁）；大型新功能才列課後。演練評分重點是**用 AI** 完成止血、判斷、溝通、交接，不是修好所有技術問題。
- 老師像桌遊主持：按時間投入事件卡，扮演顧客/總部，但**不代替學員操作**。
- 先親自跑過所有事件，確認前面單元系統至少能動。
