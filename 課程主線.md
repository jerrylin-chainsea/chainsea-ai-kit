# 課程主線 ｜ 養大你自己的 AI 小店 🏪

整門課不是三個無關的作業,而是**一條線、一間會長大的店**。每堂課都讓你的店更完整,最後變成你個人網站上的作品。

```
   M12 大腦            🏪 看得到的店         M13 對外展示          M14 經營
 ┌──────────┐  資料   ┌──────────┐  嵌入  ┌──────────┐       ┌──────────┐
 │ AI 店員的 │ ─────▶ │ 你的 AI  │ ────▶ │ 你的個人 │ ────▶ │ 經營一年 │
 │ 大腦:讀  │ 建議   │ 小店動起 │ 作品   │ 網站(公  │ 營運   │ ·處理突發│
 │ 資料給建議│        │ 來(養成) │        │ 開上線)  │       │ ·交接    │
 └──────────┘        └──────────┘        └──────────┘       └──────────┘
   run.py /            shop-showcase/       m13-astro-blog/     m14-ops-drill/
   notebook            (three.js)
```

| 模組 | 在主線裡是什麼 | 你會得到的成果 |
|---|---|---|
| **M12** | AI 店員的**大腦**:讀營運資料、抓異常、給建議 | 四欄決策(`insight / anomalies / risk / action`),notebook 跑得出來 |
| **🏪 小店** | 把大腦變成**看得到的小店**(養成感) | three.js 場景,招牌是你的店名、AI 店員講你 M12 的建議 |
| **M13** | 把小店**放上你的個人網站** | 公開網址 + 嵌入小店當作品 |
| **M14** | **經營**這間店、處理突發 | 一年營運史 + 異常處置 + 一頁交接 |

## 怎麼把這條線接起來(成果感的關鍵)

1. 跑 **M12 筆記本**(`m12-crew-starter/M12_課1_教學筆記本.ipynb`)→ 最後一格會把你的建議寫進 `shop-showcase/data.js`。
2. 打開 **🏪 小店**(`shop-showcase/`)→ 招牌變你的店名、AI 店員講的就是**你算出來的建議**。
3. 做 **M13 網站**時,把 `shop-showcase/` 嵌進你的頁面(README 有 iframe 範例)→ 變成你的線上作品。
4. **M14** 讓這間店經營一整年、撐過突發狀況。

> 每一步都是「貼 prompt → 看到自己的店有變化」。看得到變化,就學得起來,也最有成就感。

開始 → 看 [`README.md`](./README.md) 的「線上一鍵開始」。
