# M11 / U11 ｜ 16 小時 AI 程式開發實作課 · 老師劇本(教案）

> 2026-07-05 更新：NOVA WAREHOUSE 版主線請以 [`M11-16h-NOVA-授課主線.md`](./M11-16h-NOVA-授課主線.md) 為準。本檔保留作舊版劇本與細節參考。

> **這份是老師的「劇本」**：4 堂 × 4 小時 = 16 小時。學生**照劇本一步步做**，不自由發想。
> 每個動作都有:**固定 prompt → 固定操作 → 預期結果 → 驗收 → 出錯保底**。全班做出同一個結果,再往下一步。

---

## 最高原則(每一步都適用)

1. **可預期**:同一段 prompt、同一個操作,全班看到同一個結果。
2. **可驗收**:每步都有「怎麼確認做對了」——多半是**看畫面**或**看 `git diff`**。
3. **有保底**:任何一步卡住,都有 B 方案,不讓任何學生卡在原地(見每步「保底」與最後的〈通用保底〉)。
4. **人是主管,AI 是工讀生**:先讀 → 先計畫 → 加護欄 → 小範圍改 → 看 diff 驗收。順序對了才安全。

## 課程主線(一條線)

```
資料進來 → 檢查資料 → 資料合約 → AI coding agent 改功能 → Dashboard 呈現
       → LINE Flex 推播 → ops agent 自動化 → GitHub Actions → reviewer / build / commit
   ▲            C1 進得了專案        C2 管得住 AI           C3 接上平台              C4 自動化交付
```

整門課只做**一條交付線**:「倉儲營運 Dashboard + LINE OA Flex 推播 + ops agent 自動化」。海風小店是載體,主線是 **AI coding agent 驅動開發與平台整合**:
C1 把作品跑起來、先看到最終成果 → C2 用資料合約與 planner/implementer/reviewer 工作流管住 AI → C3 用 Codex/Claude Code 接 LINE OA Flex 並保留人工審核 → C4 用 CrewAI-style ops agent + GitHub Actions 自動產 artifact,再用 reviewer/build/Skill/MCP 收乾淨。

> **整門課 DoD(每堂重複)**:AI 做出來不算完成,通過驗收才算完成。驗收包含:**畫面／輸出／diff／build／human review**。

# C2 ｜ 管得住 AI(0:00–4:00)

> 目標:裝 Claude Code,認識**資料合約**(`data-lab/report.json`),學會 **planner → 人審 → implementer** 工作流;並養成內建 slash 的好習慣。
> 開場先「痛一次」:老師示範沒有護欄的 AI 亂改三個檔的 diff,再引出 AGENTS.md/CLAUDE.md 與固定流程的必要性。學生講義:`U2/STEP-01..02.md`。

## 段 1 裝上、認識面板(0:15–1:10)
- 在 VS Code 擴充功能搜 `Claude Code`(Anthropic)→ Install → 開面板 → 登入。
- **驗收**:面板能打字。**保底**:面板沒出現 → 重開 VS Code;登入轉圈 → 完成瀏覽器授權後等幾秒。

## 段 2 兩個旋鈕:模式 + 模型(1:10–2:20)
- **模式**(給 AI 多少動手權):`shift+tab` 循環 Default → Auto-Accept → **Plan**。
- **模型**:打 `/model` → 選 Opus(難題)/ Sonnet(小事)。**跑跑看**:打完 `/model` 右上「模型」標籤真的換人。
- **內建 slash 生活習慣**(本堂新增,見〈附錄 A〉):`/clear`、`/compact`、`/review`、`/status`。
- **保底**:Windows 上 `shift+tab` 有時跳過 Plan → 改用**點面板底部模式字樣**選 Plan。

## 段 3 先讀、不先改(2:20–3:05)
- **固定 prompt**:`prompts/01-read-project.md`(請先閱讀專案、不要改、回答五問)。
- **操作**:確認底部是 **Plan mode** → 貼 prompt → 送出。**驗收**:拿到摘要且 `git status` 乾淨(一個檔都沒改)。

## 段 4 planner → 人審 → implementer,安全改一次(3:05–3:55)
- **固定 prompt**:`prompts/02-plan-mode.md`(planner,只回 A–F 格式)→ 人審四題(檔案對嗎/新套件/package.json/重構傾向)→ `prompts/03-guardrails.md`(implementer,回報 A–E 格式)。
- **任務**:在 Dashboard 載入 report.json 後,新增「低庫存推播準備」小區塊。允許檔案只有 `web-lab/src/Dashboard.jsx` 與 `web-lab/src/styles.css` 的 Dashboard 區塊。
- **操作**:Plan mode 貼 planner 卡 → 看 A–F 計畫、問四題 → `shift+tab` 切離 Plan → 貼 implementer 卡 → 照它回報的 D 逐項驗收。
- **第二輪變更**:再給一個小改動,**仍走同一流程** —— 這就是「可交付」和「碰運氣」的差別。
- **驗收**:Dashboard 出現推播主題/通道/下一步、`git status` 只有允許檔案、`npm run build` 通過、`git diff` 沒動 LINE 腳本與 package.json → `git commit`。
- **保底**:AI 亂改一堆 → `git restore <file>` 還原後,加更嚴格護欄重下(見〈通用保底〉)。

**C2 收束一句**:AI 不是不能改,是**不能一開始就改**。你驗收過,才算完成。

# C3 ｜ 接上平台(0:00–4:00)

> 目標:在**營運異常 Dashboard 推播中心**,以按鈕為主走完「載入 → 檢查合約 → 視覺預覽 → 人工審核 → **按鈕推播**」,兩個範本都走(營運異常/訂單資訊),
> 再親手製造合約錯誤、用 **ReAct** 修好。沒 LINE OA 看到 `[mock]` 即過關;設好 `.env` 的同學可以真送(不再限老師)。
> 學生講義:`U3/STEP-01..03.md`。老師詳版:`u11-c3-line-oa-lab.md`。

任務素材:`web-lab/src/Dashboard.jsx`、`web-lab/vite.config.js`(dev 後端,starter 已內建)+ `data-lab/report.json`、`data-lab/orders.json` + `line-lab/sendLineAlert.js`。
**教學用固定錯誤**:把 `report.json` 的 `risk_level` 改成中文「嚴重」→ 畫面出現紅色擋牌、按推播回 `contract_error`(兩道防線、同一句錯誤)。

## 段 1 按鈕走主線 · 兩範本(0:15–1:20)
- 一鍵啟動 → 切「營運異常 Dashboard」→ 上方「推播範本」預設營運異常 → Button 1 載入 → Button 2 檢查合約(綠)→ Button 3 生成 **Flex 視覺預覽**(可切「看 JSON」)。

- **驗收**:兩範本三步都走通、說得出「畫面讀的就是 report.json / orders.json,沒有第二份資料」。
- **保底**:畫面跑不動 → 用終端機 `node line-lab/sendLineAlert.js --flex` + 看 payload 完成同段驗收。

## 段 2 Human-in-the-loop + 按鈕推播(1:20–2:10)
- 未勾 checkbox → 推播按鈕**是暗的**;認真讀過 Flex 視覺預覽 → 勾「我已人工審核這則通知內容」→ 按「推播 LINE Flex」→ 看到 `[mock] LINE_REAL_SEND is not 1, no request sent.`。
- **F12 教學(關鍵)**:Network 只有 `POST localhost:5180/api/send-line-flex`(request 無 token、無收件對象);**整個分頁沒有 api.line.me**。講清楚:瀏覽器只打自己後端,後端才帶 `.env` 的 token 去打 LINE —— 這是 production 正確樣式。token 像家裡鑰匙,不能貼在大門上。
- **想真送的同學**:填 `line-lab/.env`(token+targetId)、`LINE_REAL_SEND=1`、**重啟 dev** → 按鈕真送、手機收到、畫面顯示 `sent`。
- **固定 prompt**:`U3/PROMPT-CARD.md` 卡 1(HITL 審核,AI 只判斷不改檔)。
- **驗收**:未勾時按鈕暗;`[mock]`(或 `sent`)親眼看過;F12 只有 `/api/send-line-flex`、沒有 api.line.me。

## 休息(2:10–2:20)

## 段 3 製造合約錯 → ReAct 修好(2:20–3:20)
- 手改 `report.json`:`"risk_level": "high"` → `"嚴重"`(**改值不改語法**)→ 存檔 → 擋牌**不點擊自己出現**、Flex 預覽與推播按鈕全關;按推播回 `contract_error`(同一句)。
- **固定 prompt**:`U3/PROMPT-CARD.md` 卡 2(ReAct:Expected/Actual/Reason/Act/Observe/**Minimal Patch**/Verify/Blocker,先分析不改檔)→ 人審 Minimal Patch 是否只修合約違反 → 卡 3 放行。
- **驗收**:擋牌消失、綠色通過回來、diff 只有 `report.json` 一行。
- **保底**:JSON 改壞語法出現 Vite 全螢幕錯誤 → 改回原樣即恢復(這本身就是看錯誤訊息的練習)。

## 段 4 收尾 build/commit(3:20–3:55)
- **收尾**:`cd web-lab && npm run build` → `git status` / `git diff` / `git diff -- web-lab/package.json`(必須無輸出)→ 確認 `.env` 沒被 commit → commit。
- **安全驗收**:build 後 `grep -r "CHANNEL_ACCESS_TOKEN" web-lab/dist` 與真正的 push URL 應為空(token 與 push URL 不在前端;caption 出現 api.line.me 字樣是教學文字,無害)。
- **加做(進階組)**:Flex Message 版(`u11-c3-flex-message-lab.md`)、請 AI 擴充 Dashboard(`U3/PROMPT-CARD.md` 卡 4)。
- **驗收**:build 通過、diff 乾淨、commit 完成。

**C3 收束一句**:推播從「複製指令」變成「一顆按鈕」,而 token 一步都沒進前端 —— 你做得出功能、守得住人審、還懂為什麼要有後端。

# 附錄 A ｜ slash command 型錄

> **內建**:Claude Code 內建就有。**自建**:本課在 `.claude/commands/<name>.md` 建立(內文即 prompt,可用 `$ARGUMENTS`,frontmatter 填 `description`);Codex 亦支援自訂指令。
> 查 `/help` 看你這版所有可用指令。**注意**:Plan Mode 是「模式」(shift+tab 切),不是指令。

### A1 · plan(內建 · 模式)
- **用途**:讓 AI 只計畫、不改檔,先看它想改多少。
- **設定**:免安裝。**shift+tab** 循環到 Plan;部分版本可打 `/plan`。
- **固定操作**:切到 Plan mode → 貼需求 → 看計畫。
- **固定 prompt**:`prompts/02-plan-mode.md`。
- **預期結果**:AI 只回「會改哪個檔、哪一段、風險、驗收方式」,一個檔都沒動。
- **驗收**:`git status` 乾淨。
- **保底**:Windows shift+tab 跳過 Plan → 點面板底部模式字樣選 Plan。

### A2 · /status(內建)
- **用途**:看目前工作階段/設定狀態(與 `git status` 分工:一個看 AI 階段、一個看檔案改動)。
- **設定**:免安裝。
- **固定操作**:輸入框打 `/status`。
- **固定 prompt**:無(直接指令)。
- **預期結果**:顯示目前模型、模式、專案等狀態摘要。
- **驗收**:看得到狀態列。
- **保底**:看不懂就忽略,改看面板底部指示器 + `git status`。

### A3 · diff / 看 AI 改了什麼(內建 `/review` + 終端機 `git diff`)
- **用途**:改完後,先看它動了哪些檔、每行怎麼變。
- **設定**:`git diff` 免安裝;`/review` 內建;精緻版用自建 `/review-diff`(見 B5)。
- **固定操作**:終端機 `git diff`;或面板打 `/review`。
- **固定 prompt**:無(或用 `/review-diff`)。
- **預期結果**:綠加紅刪,只在預期範圍。
- **驗收**:沒有不相關檔案被動到。
- **保底**:`/review` 不存在此版 → 一律用 `git diff`。

### A4 · /debug(自建 → `.claude/commands/debug.md`)
- **用途**:遇到錯誤,先分析(不改)、同意後只改必要檔。
- **設定**:本課已建於 starter;內容即 `prompts/04-debug.md`。
- **固定操作**:打 `/debug` 後貼上錯誤訊息(或 `/debug <錯誤描述>`)。
- **固定 prompt**:見 `.claude/commands/debug.md`。
- **預期結果**:先給「可能原因/相關檔/如何確認/建議修法」,等你同意才改。
- **驗收**:修完後 Dashboard 擋牌消失、`node line-lab/sendLineAlert.js --flex` 恢復 mock 成功。
- **保底**:指令沒載入 → 直接貼 `prompts/04-debug.md` 全文。

### A5 · /clear、/compact(內建)
- **用途**:`/clear` 清空對話從頭開始(省 token、換任務時用);`/compact` 壓縮長對話成摘要、保留重點。
- **設定**:免安裝。
- **固定操作**:換任務前 `/clear`;對話太長、AI 開始忘記時 `/compact`。
- **固定 prompt**:無。
- **預期結果**:`/clear` 後上下文清空;`/compact` 後對話變短但重點還在。
- **驗收**:AI 不再被舊對話干擾。
- **保底**:`/compact` 後仍亂 → 直接 `/clear` 重新開始、重貼必要 context。

### A6 · commit message(自建 → `.claude/commands/commit-msg.md`)
- **用途**:根據 `git diff` 產生一句清楚的 commit 訊息(講改了什麼、為什麼)。
- **設定**:本課已建於 starter。
- **固定操作**:改完、`git add` 後打 `/commit-msg`。
- **固定 prompt**:見 `.claude/commands/commit-msg.md`(讀 staged diff、產出一行 summary + 選填說明)。
- **預期結果**:一句可直接用的 commit 訊息。
- **驗收**:訊息看得出「改了什麼」。
- **保底**:指令沒載入 → 自己寫「動詞 + 對象」如「修正 risk_level 輸出格式」。

# 附錄 C ｜ MCP 型錄(3 類)

> **共同概念**:MCP 讓 AI 在**你授權範圍內**自己去外部工具讀資料。裝前先問:能讀什麼、能不能寫、會不會碰正式資料。裝法用 Claude Code CLI `claude mcp add`,或專案層 `.mcp.json`(本課已附)。裝完打 `/mcp` 檢查。**共同保底**:連不上 → `/mcp` 看狀態、重開 VS Code;`npx` 失敗 → 確認 Node 18+;仍不行 → 退回終端機/手動,不擋進度。

### C1 · Chrome DevTools MCP(驗收 web-lab)
- **用途**:讓 AI 開真的 Chrome、看 console/network/DOM、截圖 —— 拿來驗收 `web-lab`。
- **安裝**:`claude mcp add chrome-devtools -- npx chrome-devtools-mcp@latest`(需 Node + Chrome)。或 `.mcp.json` 加一條。
- **固定操作**:先 `npm run dev` 跑起 web-lab → 請 AI「打開 localhost、看 console 有無錯誤、截一張圖」。
- **固定 prompt**:`請用 Chrome DevTools 開 http://localhost:5180,列出 console 有沒有錯誤,並截一張首頁的圖。`
- **預期結果**:AI 回報 console 狀況 + 一張截圖。
- **驗收**:`/mcp` 列出 chrome-devtools;真的拿到截圖。
- **保底**:連不上 → 自己在瀏覽器按 F12 看 console。

### C2 · Context7 MCP(查最新文件)
- **用途**:把套件**最新官方文件**餵給 AI,避免它用過時 API。
- **安裝**:`claude mcp add context7 -- npx -y @upstash/context7-mcp`(選填 `--api-key`,免費金鑰於 context7.com/dashboard;或 remote:`claude mcp add --transport http context7 https://mcp.context7.com/mcp`)。
- **固定操作**:問套件用法時,句尾加 `use context7`。
- **固定 prompt**:`用 vite 設定 dev server 埠號,請參考最新官方文件。use context7`
- **預期結果**:AI 依最新文件回答,而非記憶。
- **驗收**:`/mcp` 列出 context7;答案附上文件出處。
- **保底**:沒金鑰限流 → 改用官方網站,或改用 `/model` 換更強模型。

### C3 · Codebase Memory MCP(讓 AI 記住整個專案)
- **用途**:把程式碼庫索引成本地知識圖譜(SQLite),AI 能查**架構總覽、呼叫關係、死碼**,不用一檔一檔翻。全本地執行、**免 API key**。
- **安裝**(官方腳本會自動偵測並設定 Claude Code 等 agent):
  - Windows PowerShell:`Invoke-WebRequest -Uri https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.ps1 -OutFile install.ps1` → `.\install.ps1`
  - macOS/Linux:`curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash`
  - repo:https://github.com/DeusData/codebase-memory-mcp
- **固定操作**:先請 AI 用 `index_repository` 索引專案 → 再問架構問題。
- **固定 prompt**:`請先索引這個專案,然後告訴我:整體架構長怎樣?Dashboard.jsx 被誰使用?`
- **預期結果**:AI 用圖譜回答架構與檔案關係,不再逐檔猜。
- **驗收**:`/mcp` 列出 codebase-memory;架構問題答得出檔案關係。
- **保底**:裝不起來 → 用 `/init` 讓 AI 讀專案,或直接看 README;不擋進度。索引資料存在本機 `~/.cache/codebase-memory-mcp/`,不會上傳。
