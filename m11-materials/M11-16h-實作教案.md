# M11 / U11 ｜ 16 小時 AI 程式開發實作課 · 老師劇本(教案）

> **這份是老師的「劇本」**：4 堂 × 4 小時 = 16 小時。學生**照劇本一步步做**，不自由發想。
> 每個動作都有:**固定 prompt → 固定操作 → 預期結果 → 驗收 → 出錯保底**。全班做出同一個結果,再往下一步。

---

## 最高原則(每一步都適用)

1. **可預期**:同一段 prompt、同一個操作,全班看到同一個結果。
2. **可驗收**:每步都有「怎麼確認做對了」——多半是**看畫面**或**看 `git diff`**。
3. **有保底**:任何一步卡住,都有 B 方案,不讓任何學生卡在原地(見每步「保底」與最後的〈通用保底〉)。
4. **人是主管,AI 是工讀生**:先讀 → 先計畫 → 加護欄 → 小範圍改 → 看 diff 驗收。順序對了才安全。

## 課程主線(一條線)

```
會問 AI  ──►  指揮 AI 接手專案  ──►  改功能 / 修錯 / 驗收  ──►  用 slash / Skills / MCP 固定成 SOP
   ▲              C1 進得了專案         C2 管得住 AI        C3 做得出功能      C4 收得乾淨
```

養大你自己的 **🏖️ 海風小店**:C1 把店跑起來 → C2 安全改它 → C3 幫它做出「資料檢查」功能 → C4 把整套做法固定成工具,交接得出去。

## 四堂總覽

| 堂 | 標題 | 目標(帶得走) | 完成物 | 對應 starter 路徑 |
|---|---|---|---|---|
| **C1** | 進得了專案 | 打開 → 跑起來 → 改一個字 → 存檔 | 跑起 web-lab、改 `data.js`、第一次 `git commit` | `web-lab/`、`data-lab/` |
| **C2** | 管得住 AI | 讀→plan→護欄→小改→看 diff | 一次 Plan Mode 安全改、看懂內建 slash | `web-lab/src/data.js`、`prompts/01-03` |
| **C3** | 做得出功能 | 同一任務,Claude Code 與 Codex 都做:加規則→固定輸出→製造錯→debug | `run.py` 修好、四欄輸出正確、兩個 agent 對照 | `data-lab/run.py`、`prompts/04-05` |
| **C4** | 收得乾淨 | Git 驗收、防走鐘、把流程固定成工具 | 5 支 Skills、3 類 MCP 裝起來、SOP 交接 | `.claude/`、`.mcp.json`、`prompts/06` |

> **環境需求**:VS Code + Claude Code 擴充(C2 起)、Node 18+、Python 3、可上網。Codex 擴充於 C3 安裝。

# C2 ｜ 管得住 AI(0:00–4:00)

> 目標:裝 Claude Code,學會**先讀 → 先 plan → 加護欄 → 小範圍改 → 看 diff 驗收**;並養成內建 slash 的好習慣。

## 段 1 裝上、認識面板(0:15–1:10)
- 在 VS Code 擴充功能搜 `Claude Code`(Anthropic)→ Install → 開面板 → 登入。
- **驗收**:面板能打字。**保底**:面板沒出現 → 重開 VS Code;登入轉圈 → 完成瀏覽器授權後等幾秒。

## 段 2 兩個旋鈕:模式 + 模型(1:10–2:20)
- **模式**(給 AI 多少動手權):`shift+tab` 循環 Default → Auto-Accept → **Plan**。
- **模型**:打 `/model` → 選 Opus(難題)/ Sonnet(小事)。**跑跑看**:打完 `/model` 右上「模型」標籤真的換人。
- **內建 slash 生活習慣**(本堂新增,見〈附錄 A〉):`/clear`、`/compact`、`/review`、`/status`。
- **保底**:Windows 上 `shift+tab` 有時跳過 Plan → 改用**點面板底部模式字樣**選 Plan。

## 段 3 先讀、不先改(2:20–3:05)
- **固定 prompt**:`prompts/01-read-project.md`(請先閱讀專案、不要改、回答五問)。
- **操作**:確認底部是 **Plan mode** → 貼 prompt → 送出。**驗收**:拿到摘要且 `git status` 乾淨(一個檔都沒改)。

## 段 4 計畫 → 護欄 → 安全改一次(3:05–3:55)
- **固定 prompt**:`prompts/02-plan-mode.md`(報計畫)→ `prompts/03-guardrails.md`(護欄 + 同意修改)。
- **任務**:把標語改成「靠海的手作選物,我的第一個 AI 小店」。
- **操作**:Plan mode 看計畫 → 覺得範圍 OK → `shift+tab` 切離 Plan → 貼「同意」prompt。
- **驗收**:瀏覽器標語變了、`git status` 只有 `data.js`、`git diff` 只有 tagline 一行 → `git commit`。
- **保底**:AI 亂改一堆 → `git restore <file>` 還原後,加更嚴格護欄重下(見〈通用保底〉)。

**C2 收束一句**:AI 不是不能改,是**不能一開始就改**。

# C3 ｜ 做得出功能(0:00–4:00)

> 目標:用**同一個任務**,分別用 **Claude Code** 和 **Codex** 做出一個「資料檢查」功能:
> **新增檢查規則 → 固定四欄輸出 → 製造指定錯誤 → debug 修好**。看兩個 agent「名字不同、流程一樣」。

任務素材:`data-lab/run.py`(讀 `data/sales.csv`,印固定四欄 `insight / anomalies / risk_level / action_items`)。
`run.py` 內建一個**教學用固定 bug**:`risk_level` 被翻成中文「嚴重」,但規格只能是 `low / medium / high`。

## 段 1 看懂資料 + 固定輸出(0:15–1:20)
- 跑 `python run.py` → 看四欄輸出。看懂 CSV/JSON、負數/過大/缺漏。
- **固定 prompt**:`prompts/05-data-check.md`(固定四欄、risk_level 只能 low/medium/high、沒依據標不可採用)。
- **驗收**:輸出有四欄、anomalies 有理由、risk_level 值受限、action_items 是行動。

## 段 2 製造指定錯誤 → 看見它(1:20–1:50)
- 跑 `python run.py`,**看見** `risk_level` 印出「嚴重」——這就是要修的 bug(規格違反)。
- **驗收**:全班都看到「嚴重」。**保底**:沒 Python → 用線上 Python 或看老師畫面,重點在流程不在環境。

## 段 3 用 Claude Code 修(ReAct)(1:50–2:35)
- **固定 prompt**:`prompts/04-debug.md`(先分析、不要改 → 同意後只改必要檔)。也可直接打自建 `/debug`(見〈附錄 A〉)。
- **操作**:Plan 先想 → 看分析合理 → 放行 → 再跑 `python run.py`。
- **驗收**:`risk_level` 變 `high`;`git diff` 只動 `run.py` 那幾行(移除中文轉換)。

## 休息(2:35–2:50)

## 段 4 裝 Codex,做同一件事(2:50–3:55)
- **裝**:VS Code 擴充搜 `Codex`(OpenAI)→ Install → 登入(ChatGPT 帳號)。
- **面板**:模式 **Chat / Agent / Agent(Full Access)**;可調 **reasoning(low/med/high)** 與模型。
- **同任務**:用 `prompts/04-debug.md` 同一段,讓 Codex 修同一個 bug。Chat 先問分析 → Agent 動手。
- **對照(並列講)**:Claude Code `shift+tab` 切 Plan/Auto ↔ Codex 選 Chat/Agent;`/model` ↔ 模型/reasoning。**流程一樣**:讀 → 計畫 → 改 → 看 diff。
- **驗收**:兩個 agent 都能把 `risk_level` 修成 `high`,且只動必要檔。
- **保底**:Codex 裝不了/登入失敗 → 本段改看老師示範,學生用 Claude Code 完成即可(不擋進度)。

**C3 收束一句**:換工具不換流程。能做出功能、又修得好錯,你就從「會問」變「會做」。

# C4 ｜ 收得乾淨(0:00–4:00)

> 目標:把做法**固定成工具**,交接得出去。Git 完整驗收 → 防走鐘三症 → 建 Skills/SOP → 裝 3 類 MCP。

## 段 1 三個讓 AI 走鐘的毛病 + 一句話拉回(0:15–1:15)
| 毛病 | 症狀 | 拉回的一句話 |
|---|---|---|
| Context Drift 越改越偏 | 只要改標題,它順手改版面/資料 | 「請回到原本任務,這次**只**處理 X,其他不要改。」 |
| Prompt Debt 越補越亂 | 零碎要求補到 AI 前後矛盾 | 「請先**整理目前最新狀態**、不要改:已完成什麼、要遵守什麼、下一步只做什麼。」 |
| Regression 改 A 壞 B | 修好這個、弄壞那個 | 「請列出這次修改後需要**回歸檢查**的項目。」 |

## 段 2 Git 完整驗收(1:15–1:55)
- `git status` → `git diff` → `git add .` → `git commit`。可用自建 `/review-diff`、`/commit-msg`(見〈附錄 A〉)。
- **驗收**:diff 只有預期變更、commit 訊息講清楚「改了什麼、為什麼」。

## 段 3 把流程固定成 Skills / SOP(1:55–2:55)
- **概念**:Skill = 給新人的工作守則,貼牆上,不用每次重講。放 `.claude/skills/<name>/SKILL.md`,`/name` 呼叫,也會自動觸發。
- **本課 5 支 Skill**(逐支見〈附錄 B〉):新手專案開發、資料檢查固定輸出、debug、Git 驗收、review diff。
- **操作**:`/skills`（或 `/help`）看得到 → 對 `run.py` 跑 `data-check-fixed-output` 或 `debug-react`。
- **驗收**:Skill 觸發後,AI 照 SOP 走(先讀、固定輸出、看 diff)。

## 休息(2:55–3:05)

## 段 4 安裝與示範 MCP(3:05–3:55)
- **概念**:MCP = 幫 AI 接上外部工具的「安全的線」。裝之前先問:它**能讀什麼、能不能寫、會不會碰正式資料**。
- **本課 3 類 MCP**(逐類見〈附錄 C〉):Chrome DevTools(驗收 web-lab)、Context7(查最新文件)、Filesystem/GitHub(檔案/GitHub)。
- **操作**:裝好後打 `/mcp` 應列出;用 Chrome DevTools MCP 開 `localhost` 截圖驗收 web-lab。
- **驗收**:`/mcp` 看得到、能讓 AI 開瀏覽器看 console。**保底**:連不上 → 見〈附錄 C〉每類保底,退回終端機/手動。

**C4 收束一句**:會用一次不稀奇,能讓 AI **每次照流程做**、還能**交接**,才是真本事。

# 附錄 A ｜ slash command 型錄

> **內建**:Claude Code 內建就有。**自建**:本課在 `.claude/commands/<name>.md` 建立(內文即 prompt,可用 `$ARGUMENTS`,frontmatter 填 `description`);Codex 亦支援自訂指令。
> 查 `/help` 看你這版所有可用指令。**注意**:Plan Mode 是「模式」(shift+tab 切),不是指令。

### A1 · plan(內建 · 模式)
- **用途**:讓 AI 只計畫、不改檔,先看它想改多少。
- **設定**:免安裝。**shift+tab** 循環到 Plan;部分版本可打 `/plan`。
- **固定操作**:切到 Plan mode → 貼需求 → 看計畫。
- **固定 prompt**:`prompts/02-plan-mode.md`。
- **預期結果**:AI 只回「會改哪個檔、哪一段、風險、驗收方式」,一個檔都沒動。
- **驗收**:`git status` 乾淨。
- **保底**:Windows shift+tab 跳過 Plan → 點面板底部模式字樣選 Plan。

### A2 · /status(內建)
- **用途**:看目前工作階段/設定狀態(與 `git status` 分工:一個看 AI 階段、一個看檔案改動)。
- **設定**:免安裝。
- **固定操作**:輸入框打 `/status`。
- **固定 prompt**:無(直接指令)。
- **預期結果**:顯示目前模型、模式、專案等狀態摘要。
- **驗收**:看得到狀態列。
- **保底**:看不懂就忽略,改看面板底部指示器 + `git status`。

### A3 · diff / 看 AI 改了什麼(內建 `/review` + 終端機 `git diff`)
- **用途**:改完後,先看它動了哪些檔、每行怎麼變。
- **設定**:`git diff` 免安裝;`/review` 內建;精緻版用自建 `/review-diff`(見 B5)。
- **固定操作**:終端機 `git diff`;或面板打 `/review`。
- **固定 prompt**:無(或用 `/review-diff`)。
- **預期結果**:綠加紅刪,只在預期範圍。
- **驗收**:沒有不相關檔案被動到。
- **保底**:`/review` 不存在此版 → 一律用 `git diff`。

### A4 · /debug(自建 → `.claude/commands/debug.md`)
- **用途**:遇到錯誤,先分析(不改)、同意後只改必要檔。
- **設定**:本課已建於 starter;內容即 `prompts/04-debug.md`。
- **固定操作**:打 `/debug` 後貼上錯誤訊息(或 `/debug <錯誤描述>`)。
- **固定 prompt**:見 `.claude/commands/debug.md`。
- **預期結果**:先給「可能原因/相關檔/如何確認/建議修法」,等你同意才改。
- **驗收**:修完 `python run.py` 的 `risk_level` 變 `high`。
- **保底**:指令沒載入 → 直接貼 `prompts/04-debug.md` 全文。

### A5 · /clear、/compact(內建)
- **用途**:`/clear` 清空對話從頭開始(省 token、換任務時用);`/compact` 壓縮長對話成摘要、保留重點。
- **設定**:免安裝。
- **固定操作**:換任務前 `/clear`;對話太長、AI 開始忘記時 `/compact`。
- **固定 prompt**:無。
- **預期結果**:`/clear` 後上下文清空;`/compact` 後對話變短但重點還在。
- **驗收**:AI 不再被舊對話干擾。
- **保底**:`/compact` 後仍亂 → 直接 `/clear` 重新開始、重貼必要 context。

### A6 · commit message(自建 → `.claude/commands/commit-msg.md`)
- **用途**:根據 `git diff` 產生一句清楚的 commit 訊息(講改了什麼、為什麼)。
- **設定**:本課已建於 starter。
- **固定操作**:改完、`git add` 後打 `/commit-msg`。
- **固定 prompt**:見 `.claude/commands/commit-msg.md`(讀 staged diff、產出一行 summary + 選填說明)。
- **預期結果**:一句可直接用的 commit 訊息。
- **驗收**:訊息看得出「改了什麼」。
- **保底**:指令沒載入 → 自己寫「動詞 + 對象」如「修正 risk_level 輸出格式」。

# 附錄 C ｜ MCP 型錄(3 類)

> **共同概念**:MCP 讓 AI 在**你授權範圍內**自己去外部工具讀資料。裝前先問:能讀什麼、能不能寫、會不會碰正式資料。裝法用 Claude Code CLI `claude mcp add`,或專案層 `.mcp.json`(本課已附)。裝完打 `/mcp` 檢查。**共同保底**:連不上 → `/mcp` 看狀態、重開 VS Code;`npx` 失敗 → 確認 Node 18+;仍不行 → 退回終端機/手動,不擋進度。

### C1 · Chrome DevTools MCP(驗收 web-lab)
- **用途**:讓 AI 開真的 Chrome、看 console/network/DOM、截圖 —— 拿來驗收 `web-lab`。
- **安裝**:`claude mcp add chrome-devtools -- npx chrome-devtools-mcp@latest`(需 Node + Chrome)。或 `.mcp.json` 加一條。
- **固定操作**:先 `npm run dev` 跑起 web-lab → 請 AI「打開 localhost、看 console 有無錯誤、截一張圖」。
- **固定 prompt**:`請用 Chrome DevTools 開 http://localhost:5173,列出 console 有沒有錯誤,並截一張首頁的圖。`
- **預期結果**:AI 回報 console 狀況 + 一張截圖。
- **驗收**:`/mcp` 列出 chrome-devtools;真的拿到截圖。
- **保底**:連不上 → 自己在瀏覽器按 F12 看 console。

### C2 · Context7 MCP(查最新文件)
- **用途**:把套件**最新官方文件**餵給 AI,避免它用過時 API。
- **安裝**:`claude mcp add context7 -- npx -y @upstash/context7-mcp`(選填 `--api-key`,免費金鑰於 context7.com/dashboard;或 remote:`claude mcp add --transport http context7 https://mcp.context7.com/mcp`)。
- **固定操作**:問套件用法時,句尾加 `use context7`。
- **固定 prompt**:`用 vite 設定 dev server 埠號,請參考最新官方文件。use context7`
- **預期結果**:AI 依最新文件回答,而非記憶。
- **驗收**:`/mcp` 列出 context7;答案附上文件出處。
- **保底**:沒金鑰限流 → 改用官方網站,或改用 `/model` 換更強模型。

### C3 · Filesystem / GitHub MCP(檔案 / GitHub)
- **用途**:Filesystem 讓 AI 在**指定資料夾**內讀寫檔;GitHub 讓 AI 讀 issue/PR/repo。
- **安裝**:
  - Filesystem:`claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem <允許目錄>`(把 `<允許目錄>` 設成專案根,**只給這個資料夾**)。
  - GitHub(remote HTTP,需 PAT):`claude mcp add --transport http github https://api.githubcopilot.com/mcp -H "Authorization: Bearer <PAT>"`(PAT 於 github.com/settings/tokens,scope 最小 `repo` + `read:org`)。
- **固定操作**:Filesystem 問「這個資料夾有哪些檔」;GitHub 問「列出這個 repo 的 open issues」。
- **固定 prompt**:`列出 data-lab/ 底下的檔案,並說明每個檔用途。`（Filesystem）
- **預期結果**:AI 只在授權目錄內動作;GitHub 能列出 issue。
- **驗收**:`/mcp` 列出;Filesystem 動不了授權外的檔。
- **保底**:GitHub PAT 麻煩 → 課堂只示範 Filesystem;PAT 千萬別提交進 Git。

# 通用保底(任何一步都適用)

| 狀況 | B 方案 |
|---|---|
| AI 亂改一堆 | `git status` 看範圍 → `git restore <檔>` 或 `git checkout -- <檔>` 還原 → 加更嚴格護欄重下 |
| `npm install` / `npx` 失敗 | 確認 Node 18+(`node -v`);網路;仍不行看老師畫面,流程不中斷 |
| Python 跑不動 | 用線上 Python 或看老師示範,重點在 debug 流程 |
| Claude Code / Codex 連不上 | 重開 VS Code、重新登入;仍不行改用網頁版整理想法,動手部分看示範 |
| MCP 連不上 | `/mcp` 看狀態、重開;退回終端機/手動 |
| shift+tab 跳過 Plan(Windows) | 點面板底部模式字樣選 Plan,或打 `/plan` |
| 全班進度落差大 | 老師每 10–15 分對齊一次;落後者用保底方案跟上,不卡整班 |
