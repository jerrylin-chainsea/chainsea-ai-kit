## 第二階段：Git & GitHub



**時間：80 分鐘**



這一段的目標是：



> 把 RAR 解壓後的教材專案，初始化成 Git 專案，並 push 到學生自己的 GitHub repo。



### 1. 先講 Git / GitHub 是什麼



要先講清楚：



* Git：本機版本紀錄

* GitHub：雲端倉庫

* Git 不是拿教材的工具

* GitHub 不是壓縮檔上傳區

* Git 是用來追蹤「解壓後的專案檔案變化」



這句要講清楚：



> RAR 是老師發教材的方法，Git 是學生保存開發紀錄的方法。



### 2. 設定 Git 帳號



教：



```bash

git config --global user.name "你的名字"

git config --global user.email "你的 GitHub Email"

```



這裡不用太深，只要讓學生知道：



> Git commit 需要知道這筆紀錄是誰做的。



### 3. 到 GitHub 開 repo



帶學生：



* 登入 GitHub

* 建立新 repository

* repo 可以叫 `web-lab` 或 `ai-course-web-lab`

* 不要勾 README

* 複製 repo URL



### 4. 初始化 Git



在 VSCode terminal 裡，站在正確資料夾後：



```bash

git init

git status

```



這裡要講：



* `git init`：讓這個資料夾開始被 Git 管理

* `git status`：查看現在有哪些檔案還沒被追蹤



### 5. 修改檔案，教 Ctrl + S



例如改 `src/data.js`：



* 改店名

* 按 `Ctrl + S`

* 看 localhost 畫面是否更新



這裡你會順便講：



> VSCode 裡改了檔案，不代表真的保存，看到分頁小白點要記得 Ctrl + S。



### 6. 講 Git 狀態



用這一組概念：



* untracked：Git 還沒開始追蹤

* modified：已追蹤檔案被修改

* staged：準備提交

* commit：本機版本節點

* push：上傳到 GitHub



基本指令：



```bash

git status

git add .

git commit -m "init project"

git remote add origin <你的 GitHub repo URL>

git branch -M main

git push -u origin main

```



### 7. 帶過 fetch / pull / branch / tag / git flow



這裡建議只講概念，不要全部實作。



你可以這樣定位：



* `git fetch origin`：先看看雲端有什麼新東西

* `git pull`：把雲端更新拉回來

* branch：開一條安全分支，不要直接弄壞 main

* tag：幫重要版本插旗

* git flow：多人開發時，通常是開分支、提交、PR、review、merge



課 1 不要讓學生真的跑完整 Git flow，先知道真實工作長什麼樣子就好。

## 第三階段：Claude Code / Codex 基本操作



**時間：50 分鐘**



這一段的目標是：



> 學生知道 Claude Code / Codex 不是聊天視窗，而是可以進入專案的 coding agent，而且要用 slash command、模式與權限控制它。



### 1. Claude Code 插件登入



你會先讓學生打個 `hello`。



學生可能會看到：



* 尚未登入

* 插件引導登入

* 登入完成後可以開始使用



這段不需要太久，重點是讓學生知道：



> 第一次使用通常會先登入授權，這是正常流程。



### 2. 模型簡介



簡單比：



* Haiku：快、省

* Sonnet：平衡，日常主力

* Opus：強，但較慢、較花費

* 其他模型名稱依工具當天 UI 為準



建議不要花太多時間講模型歷史，直接用「快 / 穩 / 強 / 貴」的角度說就好。



### 3. Claude Code slash command



你會教：



* `/model`：切換模型

* `/effort`：調整思考深度

* `/usage`：查看用量

* `/compact`：壓縮上下文

* `/goal`：設定任務目標

* `/btw`：插入問題



每個都講應用場景，不要求背。



### 4. Shift + Tab 模式切換



要講：



* Plan mode：先規劃，不改檔

* Ask before edits：改之前先問

* Edit automatically：比較快，但要會看 diff



這裡要回扣 Git：



> AI 動手前最好先 commit 或開 branch，AI 動手後一定要看 diff。



### 5. Codex 插件



Codex 不用重教一輪，而是對照 Claude Code。



你會教：



* `/status`：看設定、權限、用量

* `/goal`：設定或清除任務目標

* `/model`：換模型與推理等級

* `/approve`：核准被擋下的動作

* `/compact`：壓縮上下文

* `/plan`：進入規劃模式



也要講 shift+tab / 模式切換。



這段重點是：



> 工具不同，但控制邏輯相同：先看狀態、先定目標、先規劃、再核准、最後驗收。
